\# v0.3.0 - Identity Governance Engine



\## Overview



This release introduces the Identity Governance Engine for BlackKnight One.



The focus of this release is governed access: how access is requested, approved, assigned, validated, reviewed, expired, and reported.



\## Added



\- Identity Governance Engine documentation

\- Access Package framework

\- Entitlement Management design

\- Administrative Unit planning

\- Dynamic Group planning

\- Access Review planning

\- Lifecycle Workflow planning

\- GDAP-inspired governance model



\## Core Concepts



\- Access Packages

\- Entitlement Management

\- Access Reviews

\- Administrative Units

\- Dynamic Groups

\- Lifecycle Workflows

\- Delegated access governance



\## BlackKnight Principle



Access should never be assumed.



Access should be requested, approved, validated, reviewed, and backed by evidence.
